# Object-Following-Robot
An object-following robot uses a camera connected to MATLAB, where the user selects the target via ROI in the first frame. The system tracks the object independent of shape or color and sends motion commands to a microcontroller to drive the motors. If tracking fails or speed is too high, the robot automatically returns to its start position.


An object-following robot can be designed to operate independently of the object’s shape and color by initializing tracking through a user-defined Region of Interest (ROI) selected from the first captured image frame. Once the ROI is selected, the system extracts robust visual features (orrelation-based templates) from that region and continuously tracks the object across subsequent frames using a camera interfaced with MATLAB. The tracking algorithm estimates the object’s centroid displacement and motion trajectory, allowing MATLAB to compute control commands (e.g., linear and angular velocity) based on visual servoing principles. These control signals are transmitted to a microcontroller (ATMEGA/ Arduino), which actuates the motor drivers to adjust wheel speeds and orient the robot toward the moving target. If the object’s speed exceeds the robot’s mechanical limits or tracking confidence drops below a predefined threshold (due to occlusion, abrupt motion, or loss of features), a fail-safe mechanism is triggered, causing the robot to autonomously return to its predefined starting position using stored positional references or dead-reckoning. This architecture requires a single camera for vision acquisition, MATLAB for image processing and control computation, and a microcontroller-motor interface for real-time actuation, enabling robust, adaptive object-following behavior.
